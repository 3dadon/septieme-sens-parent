# Septième Sens — Workspace global

## Vision

Septième Sens est un projet artistique, poétique et sensoriel.

Le projet explore ce qui donne du goût à la vie à travers les sens :

- Vue
- Ouïe
- Odorat
- Goût
- Toucher
- Esprit / intuition

Le site doit devenir une galerie digitale sensible : images, récits, matières, sons, expériences, IA de découverte et parcours sensoriels.

## Objectif du workspace

Organiser proprement les projets :

- frontend web
- backend API
- mobile futur
- documentation
- assets de marque
- expérimentations IA

## Philosophie produit

Priorités :

1. beauté simple
2. expérience mobile-first
3. MVP publié rapidement
4. architecture légère
5. évolution progressive vers app mobile
6. IA utile, poétique et non gadget

Éviter :

- sur-architecture
- refactors massifs non demandés
- complexité prématurée
- dépendances inutiles
- design SaaS / startup
- perte de la direction artistique

## Direction artistique

Le projet doit rester :

- très aéré
- éditorial
- luxueux sans ostentation vulgaire
- contemplatif
- mobile-first
- fluide
- imagé
- minimal en texte

Références d’ambiance :

- galerie artistique
- magazine luxe
- ETQ Amsterdam
- grandes images
- typographie élégante
- beige chaud, sable, blanc cassé, or doux, noir profond

## Identité

Reprendre l’esprit du logo Septième Sens :

- chiffre romain VII
- feuille d’olivier
- icônes des sens
- capsule diffuse
- élégance sensorielle

Si les assets ne sont pas disponibles, utiliser temporairement un monogramme `VII`.

## Règles pour agents IA

- Intervention chirurgicale uniquement.
- Respecter strictement le périmètre demandé.
- Proposer un plan minimal avant modification si la tâche est large.
- Ne pas refactorer sans demande explicite.
- Ne pas casser la DA existante.
- Ne pas introduire de dépendances lourdes sans justification.
- Toujours expliquer les fichiers modifiés.
- Stopper en cas de boucle d’erreur, lock fichier ou problème environnemental.
- Prioriser un résultat publiable plutôt qu’un système parfait.
- Préserver la possibilité d’une future app mobile.

## Workflow attendu

Le workspace doit permettre de lancer rapidement :

- frontend web
- backend API
- mobile futur
- éventuels services IA

Les commandes importantes doivent rester documentées et simples.

## Git & qualité

- Commits petits et explicites.
- Une tâche = un périmètre clair.
- Ne modifier que les fichiers nécessaires.
- Vérifier build/lint uniquement quand pertinent.
- Préserver la stabilité de la V1.
- Favoriser du code lisible, simple et maintenable.

## Mobile-first

Le trafic viendra probablement des réseaux sociaux.

Le web doit donc être excellent sur mobile :

- navigation au pouce
- images bien cadrées
- textes courts
- performance correcte
- animations légères
- responsive desktop ensuite

## Capitalisation

Quand une découverte technique, design ou IA est importante, proposer une note de capitalisation.

Ne pas l’insérer automatiquement en base ou dans un système externe sans demande explicite.
